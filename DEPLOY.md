# Инструкция по развёртыванию на REG.RU (Apache + PHP 8.2 + MySQL 8)

## 1. Подготовка на хостинге

### 1.1 Создание базы данных MySQL

1. Войдите в панель управления REG.RU
2. Перейдите в раздел **Базы данных MySQL**
3. Нажмите **Создать базу данных**
4. Запишите:
   - Имя базы данных (например: `u1234567_logoped`)
   - Имя пользователя (например: `u1234567_logoped`)
   - Пароль базы данных
5. Убедитесь что пользователь имеет **все привилегии** на базу

### 1.2 Загрузка файлов

1. Подключитесь к хостингу по **FTP** или через **Файловый менеджер** в панели REG.RU
2. Перейдите в папку `public_html` (или `www` — корневая папка сайта)
3. Загрузите **все файлы и папки** из этого архива в корень сайта:

```
public_html/
├── api/
│   ├── admin/
│   │   ├── applications.php
│   │   ├── hero-images.php
│   │   ├── login.php
│   │   ├── logout.php
│   │   ├── me.php
│   │   ├── notification-settings.php
│   │   ├── register.php
│   │   ├── schedule.php
│   │   └── session.php
│   ├── config.php        ← ЗАПОЛНИТЬ!
│   ├── db.php
│   ├── helpers.php
│   ├── hero-images.php
│   ├── booked-slots.php
│   ├── create-booking.php
│   ├── work-schedule.php
│   ├── init-db.php        ← УДАЛИТЬ после инициализации!
│   └── schema.sql
├── assets/               ← CSS и JS (Vite build)
├── images/               ← Изображения сайта
├── uploads/
│   └── hero/             ← Загруженные hero-изображения
├── .htaccess
├── index.html
├── favicon.png
└── favicon.ico
```

### 1.3 Настройка config.php

Откройте файл `api/config.php` и заполните данные БД:

```php
$DB_HOST = 'localhost';
$DB_NAME = 'u1234567_logoped';      // ← ваше имя БД
$DB_USER = 'u1234567_logoped';      // ← ваш пользователь
$DB_PASS = 'ваш_пароль_от_БД';     // ← ваш пароль
$DB_DRIVER = 'mysql';
```

При необходимости измените `$EMAIL_TO` на свой email.

### 1.4 Инициализация базы данных

1. Откройте в браузере: `https://ваш-домен.ru/api/init-db.php`
2. Вы должны увидеть: `{"message":"Database initialized successfully"}`
3. **ВАЖНО: Сразу удалите файл `api/init-db.php`** через FTP или файловый менеджер (он больше не нужен и является угрозой безопасности)

### 1.5 Создание первого администратора

1. Откройте `https://ваш-домен.ru/admin/login`
2. Поскольку администраторов ещё нет, появится форма **регистрации**
3. Введите email и пароль (минимум 6 символов)
4. Нажмите **Создать учётную запись**
5. После этого регистрация **автоматически закроется** — новых администраторов создать через сайт будет нельзя

### 1.6 Проверка работы API

Проверьте каждый endpoint в браузере:

| Endpoint | Ожидаемый результат |
|----------|-------------------|
| `/api/work-schedule.php` | JSON-массив с 7 днями недели |
| `/api/booked-slots.php` | Пустой JSON-массив `[]` |
| `/api/hero-images.php` | Пустой JSON-массив `[]` |

### 1.7 Права на папки

Установите права через FTP или SSH:

```
chmod 755 uploads/
chmod 755 uploads/hero/
chmod 755 images/
```

На REG.RU обычно права ставятся автоматически, но если загрузка фото не работает — проверьте права.

## 2. Восстановление доступа к админке

### 2.1 Сброс пароля через phpMyAdmin

1. Войдите в панель REG.RU → **phpMyAdmin**
2. Выберите вашу базу данных
3. Откройте таблицу `admin_users`
4. Найдите строку с нужным email
5. Нажмите **Изменить** (Edit)
6. В поле `password_hash` вставьте новый хэш (см. ниже)
7. Нажмите **Вперёд** (Go)

### 2.2 Генерация нового хэша пароля

Создайте временный файл `api/reset-password.php`:

```php
<?php
echo password_hash('ваш_новый_пароль', PASSWORD_BCRYPT);
```

Откройте его в браузере, скопируйте хэш, вставьте в phpMyAdmin, затем **удалите файл**.

### 2.3 SQL-запрос для сброса пароля

В phpMyAdmin выполните SQL:

```sql
-- Сначала сгенерируйте хэш через PHP (см. выше), затем:
UPDATE admin_users
SET password_hash = '$2y$10$ВСТАВЬТЕ_СЮДА_ХЭШ'
WHERE email = 'admin@example.ru';
```

### 2.4 Создание нового администратора вручную

В phpMyAdmin выполните:

```sql
INSERT INTO admin_users (email, password_hash)
VALUES ('new_admin@example.ru', '$2y$10$ВСТАВЬТЕ_СЮДА_ХЭШ');
```

Хэш генерируется через временный PHP-файл (см. п. 2.2).

## 3. Управление контентом

### 3.1 Hero-изображения

- **Загрузка**: Админка → Панель управления → блок «Изображения Hero-секции»
- **Формат URL**: укажите URL вида `/images/photo-hero.jpg` или `/uploads/hero/filename.jpg`
- **Хранение**: загруженные через админку файлы сохраняются в `uploads/hero/`
- **Права**: папка `uploads/hero/` должна иметь права `755`
- **Размер**: рекомендуется до 5 МБ, форматы JPG, PNG, WebP

### 3.2 Расписание

1. Войдите в админку: `https://ваш-домен.ru/admin/login`
2. Перейдите в **Расписание**
3. Для каждого дня недели:
   - Включите/выключите переключатель (рабочий/выходной)
   - Укажите время начала и конца приёма
   - Укажите длительность слота (в минутах)
4. Нажмите **Сохранить расписание**
5. Изменения сразу отразятся в календаре на главной странице

### 3.3 Резервная копия базы данных

#### Через phpMyAdmin:
1. Откройте phpMyAdmin
2. Выберите базу данных
3. Нажмите **Экспорт**
4. Формат: SQL
5. Нажмите **Вперёд** — скачается файл `.sql`

#### Через SSH (если доступен):
```bash
mysqldump -u пользователь -p имя_базы > backup_$(date +%Y%m%d).sql
```

#### Восстановление из бэкапа:
```bash
mysql -u пользователь -p имя_базы < backup_20250301.sql
```

Или через phpMyAdmin: вкладка **Импорт** → выбрать файл `.sql`

## 4. Устранение неполадок

### Календарь не показывает рабочие дни
- Проверьте `/api/work-schedule.php` в браузере
- Убедитесь что в таблице `work_schedule` есть записи с `is_working_day = 1`
- Проверьте консоль браузера (F12) на ошибки

### Кнопка «Сохранить расписание» не работает
- Проверьте что CSRF-токен передаётся (заголовок X-CSRF-Token)
- Попробуйте выйти и войти заново в админку
- Проверьте консоль браузера на ошибки 403

### Ошибка 500 на API
- Проверьте `api/config.php` — правильные ли данные БД
- Проверьте что PHP 8.2 активирован
- Проверьте что расширение `pdo_mysql` включено

### SPA-маршруты возвращают 404
- Проверьте что файл `.htaccess` загружен в корень
- Проверьте что `mod_rewrite` включён (на REG.RU обычно включён по умолчанию)
- В панели REG.RU: Сайты → ваш домен → Тип Apache: должен быть «Apache + PHP 8.2»

## 5. Безопасность

- Файл `api/config.php` содержит пароль от БД — убедитесь что он недоступен напрямую
- После инициализации БД **обязательно удалите** `api/init-db.php`
- Регулярно делайте резервные копии базы
- Не храните пароли в открытом виде
- Сессия администратора истекает через 30 минут неактивности
- Защита от перебора: после 5 неудачных попыток входа — блокировка на 15 минут
